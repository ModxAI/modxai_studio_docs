# 频道连接

> v1.1.0
> 2026-05

频道连接允许您通过外部即时通讯平台（Telegram、Discord、钉钉、飞书）的机器人，与本地智能体服务进行交互。您可以在任何地方通过聊天消息向智能体发送任务、切换会话、调整配置 —— 就像在 ModxAI Studio 中一样。

## 支持平台

| 平台 | 凭据要求 | 配对方式 |
|------|---------|---------|
| **Telegram** | Bot Token | `/start <配对码>` |
| **Discord** | Bot Token | `/start <配对码>` |
| **钉钉** | Client ID + Client Secret | `/start <配对码>` |
| **飞书** | App ID + App Secret | `/start <配对码>` |

## 快速上手

### 第一步：配置平台凭据

1. 在目标平台（如 Telegram）创建机器人，获取 Bot Token
2. 打开频道连接页面，在「聊天机器人」中选择对应平台
3. 填入凭据（Bot Token 或 App ID + App Secret）
4. 点击**保存设置**

### 第二步：启动服务

点击**启动连接**。系统会：

1. 检查凭据是否有效
2. 启动网关子进程
3. 注册机器人适配器
4. 启动后显示服务状态为「已连接」

### 第三步：配对设备

启动后页面会显示配对指令，格式为：

```
/start <配对码>
```

在目标平台中向机器人发送上述指令，完成配对。配对后该聊天/频道/会话即可与智能体交互。

### 第四步：开始对话

配对成功后，在平台中直接发送消息即可触发智能体响应。响应会遵循您在智能体工作台中配置的角色、模型和工具策略。

## 配对与安全

### 配对流程

1. 在频道连接页面启动服务后，系统自动生成**配对码**
2. 页面显示配对指令 `/start <配对码>`，点击**复制**按钮复制
3. 在目标平台向机器人发送 `/start <配对码>`
4. 配对成功后，机器人回复确认消息
5. 该聊天/频道/会话被加入允许列表，此后可直接发送消息

> **安全机制**：只有配对成功的来源才能向智能体发送消息。未配对的来源向机器人发消息会被忽略。

## 控制指令

在配对后的平台中，您可以向机器人发送以 `/` 开头的指令来控制行为。指令分为**公共指令**和**管理员指令**两类。
传参要求: `<必填项>` `[可选项]` 

### 公共指令

以下指令无需管理员权限即可使用：

| 指令 | 说明 |
|------|------|
| `/help` | 显示所有可用指令的帮助 |
| `/status` | 显示当前会话状态（角色、模型、工具配置等） |
| `/stop` | 停止当前正在生成的回复 |
| `/list` | 列出当前控制源下的所有子会话 |
| `/list_roles` | 列出所有角色及其模型配置 |
| `/list_models` | 列出可用的本地和在线模型 |
| `/list_tools [role]` | 列出工具组及其启用状态，可指定角色 |

### 管理员指令

使用 `/admin <密码>` 进入管理员模式后可执行：

**会话管理：**

| 指令 | 说明 |
|------|------|
| `/new [name]` | 创建新子会话并切换过去 |
| `/change <编号\|名称>` | 切换到指定子会话 |
| `/rename <名称>` | 重命名当前会话 |
| `/delete <名称>` | 删除指定子会话 |
| `/clear` | 清空当前会话上下文 |

**参数配置：**

| 指令 | 说明 |
|------|------|
| `/model <编号\|Provider/Model> [role]` | 为角色设置模型 |
| `/system [role] <文本>` | 设置角色的系统提示词 |
| `/temperature <数值> [role]` | 设置温度参数 |
| `/max_tokens <数值> [role]` | 设置最大输出 Token 数 |
| `/thinking <on\|off> [role]` | 开启/关闭思考模式 |
| `/role_timeout <秒> [role]` | 设置推理超时时间 |
| `/tools_rounds <数值> [role]` | 设置工具调用最大轮次 |

**工具控制：**

| 指令 | 说明 |
|------|------|
| `/switch_tools <on\|off> [role]` | 开启/关闭工具调用 |
| `/auto_tools <on\|off> [role]` | 开启/关闭自动工具选择 |
| `/tools <编号\|all> <on\|off> [role]` | 按工具组开启/关闭工具 |
| `/allow_risky_tools <on\|off>` | 切换高风险工具策略 |

**工作区与运行：**

| 指令 | 说明 |
|------|------|
| `/dir <相对路径>` | 设置工作区子目录 |
| `/yolo <on\|off>` | 开启/关闭自动审批 |
| `/switch_role <summary\|memory\|security\|evaluate> <on\|off>` | 切换内置角色 |
| `/approve` | 批准当前等待审批的操作 |
| `/reject` | 拒绝当前等待审批的操作 |

### 管理员认证

管理员认证为**进程内缓存**，网关服务重启后需要重新认证。

| 指令 | 说明 |
|------|------|
| `/admin <密码>` | 输入管理员密码进入管理员模式 |
| `/admin exit` | 退出管理员模式 |

> **首次设置**：如果未配置管理员密码，系统会在首次启动时自动生成一个不可见密码。您可在「更多设置」中修改管理员密码。

## 会话管理（管理员）

### 子会话机制

每个外部平台来源（聊天/频道/会话）可以管理**多个子会话**：

- 默认会话：配对后自动创建，直接收发消息
- 子会话：通过 `/new` 创建，可独立配置和切换
- `/list` 列出所有子会话，按编号切换

### 惰性启动

通过 `/new` 创建的子会话会先保存配置（角色、模型、工具等），在发送第一条正文消息时才真正创建智能体会话。您可以在新建后先用 `/model`、`/system` 等指令完成配置。

### 模型设置

- `/model <编号>` 从本地模型列表中选择模型
- `/model <Provider/Model>` 使用在线模型，如 `deepseek/deepseek-chat`
- 可选的 `[role]` 参数指定角色，不指定则作用于主角色

> **注意**：记忆角色必须使用本地模型，不支持在线或局域网方式。

## 高级设置

在「更多设置」折叠面板中可配置以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 监听地址 | `0.0.0.0` | 网关服务绑定的地址 |
| 本机端口 | `18092` | 网关服务端口，范围 1024-65535 |
| 回复等待时间（秒） | `600` | 等待智能体响应的超时时间 |
| 启动超时（秒） | `60` | 网关服务健康检查等待时间 |
| 管理员密码 | 自动生成 | 用于 `/admin` 指令认证 |

> **安全建议**：如果仅本机使用，可将监听地址设置为 `127.0.0.1`。如需局域网访问，保留 `0.0.0.0`。

## 注意事项

### 服务依赖

- 频道服务以**独立子进程**运行，依赖 ModxAI 主服务已启动
- 停止主服务时，频道服务也会随之终止
- 必须为启用的平台提供有效凭据，否则启动会失败

### 凭据安全

- 凭据（Token、Secret）在保存后**输入框自动清空**，不在页面长期保留明文
- 页面显示的是凭据的掩码（如 `****abc123`），不会回显完整凭据
- 如需更换凭据，直接在输入框中填写新凭据并保存

### 工具策略

- 频道会话的**高风险工具策略**与 Studio 设置保持一致
- 通过 `/allow_risky_tools` 切换后，会影响当前控制源下的**所有子会话**
- `block_high_risk_tools` 启用时，工具允许列表会自动过滤高风险工具

### 连接状态

- 页面顶部展示当前服务状态（已连接 / 未启动）
- **刷新**按钮重新获取网关运行状态和配置
- **重置**按钮将表单恢复到当前保存的配置

### 多平台

同一时间只能启用**一个**机器人平台。如需切换平台，选择新平台后保存并重新启动即可。
