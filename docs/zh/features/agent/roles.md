## 角色概览

| 角色 | 核心职责 | 默认状态 | 工具权限 |
|------|---------|---------|---------|
| 主角色 | 推理、工具编排、最终回复 | 必须启用 | 全部可选 |
| 评估角色 | 审查回复质量、提供反馈 | 默认启用 | 默认有限工具 |
| 摘要角色 | 上下文压缩、辅助总结 | 默认关闭 | 默认有限工具 |
| 记忆角色 | 模糊记忆搜索（微调模型） | 默认关闭 | 禁止使用工具 |
| 安全角色 | 高风险内容前置审查 | 默认关闭 | 默认禁止使用工具 |
| 委派角色 | 执行委派的子任务 | 按需添加 | 可独立配置 |

## 主角色

**主角色是唯一必须配置的角色**。未配置主角色模型时，无法创建新会话或发送消息。

### 核心职责

- 理解用户任务目标，拆解执行步骤
- 选择合适的工具完成任务
- 生成最终回复
- 根据需要将子任务委派给其他角色

### 连接方式

主角色支持三种模型连接方式：

| 方式 | 说明 | 配置要点 |
|------|------|---------|
| **本地** | 使用已导入模型库的本地模型 | 选择模型类别 → 选择具体模型 |
| **在线** | 通过 API Key 连接云端服务商 | 选择提供方 → 绑定 API 凭据 → 选择模型 |
| **局域网** | 连接局域网内的模型服务 | 输入服务地址 |

### 推荐模型

- **Qwen3.5 系列**（9B+）：推荐首选，工具调用和指令遵循能力强
- 参数至少 4B，上下文窗口建议 32K 以上
- 量化格式建议 Q4_K_M 及以上

### 默认推理参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 温度 | 0.6 | 控制回答随机性 |
| 最大生成长度 | 0（自动） | 0 表示按上下文 30% 自动计算 |
| 最大工具轮次 | 100 | 单次任务最大工具调用轮数 |
| 推理超时 | 600 秒 | 单次推理最大等待时间 |
| 流式输出 | 开启 | 实时显示生成内容 |
| 思考模式 | 开启 | 支持模型生成原生思考块 |

## 评估角色

**默认启用**，负责审查主角色的正式回复质量。

### 核心职责

- 评估回复内容是否满足用户需求
- 检查回复的逻辑完整性和事实准确性
- 提出修改建议或要求重新生成
- 确认回复质量达标后放行

### 配置要点

- 默认启用，可在角色参数中关闭
- 关闭后，主角色回复不再经过质量审查
- 推理参数偏保守（温度 0.6），确保评估一致性
- 工具轮次限制为 5 轮
- 不支持流式输出
- 不指定模型时自动使用主角色模型

## 摘要角色

**默认关闭**，用于上下文压缩与辅助总结。

### 核心职责

- 当对话长度接近模型上下文窗口时，压缩历史消息
- 生成可继续使用的上下文摘要
- 降低长对话的内存和计算开销

### 配置要点

- 推荐使用 HF\Online\LAN 标准模型（如小型摘要模型）
- 默认温度 0.4，偏保守以确保摘要质量
- 不支持流式输出
- 建议长任务场景下启用
- 不指定模型时自动使用主角色模型

---

## 记忆角色

**默认关闭**，用于基于微调记忆模型的模糊记忆搜索。

### 核心职责
- 辅助主角色进行快速模糊记忆匹配

### 重要约束

> ⚠️ 记忆角色有以下硬性限制：
> - **只能使用本地模型**，不支持在线或局域网
> - **禁止使用工具**（工具轮次固定为 0）
> - **不支持流式输出**
> - 需要专门微调的记忆模型才能发挥最佳效果
> - 支持加载 LoRA 适配器

### 调用方式

记忆角色通过被主角色按需调用。用户无需手动操作，启用后智能体会自动在需要时查询记忆。

## 安全角色

**默认关闭**，用于高风险内容的前置审查。

### 核心职责

- 审查用户输入是否包含提示注入
- 检测工具输出中的恶意代码或敏感信息
- 对 Web 获取的外部内容进行风险评估

### 安全审查机制

系统采用**两层安全检测**：

1. **纯逻辑检测**：脏词过滤、结构破坏识别
2. **安全角色模型检测**：启用安全角色后，额外使用模型进行深度审查

### 配置要点

- 启用后，高风险工具会先经过安全审查
- 检测到风险时暂停执行，等待用户确认
- 默认温度 0.2，最大生成长度 2048
- 不支持流式输出
- 不指定模型时自动使用主角色模型

## 自定义角色

需通过自定义角色系统提示词明确角色职责定位, 主角色会按需将子任务分派给独立配置的角色执行。或明确通过**角色名称**要求主角色主动调用.

### 使用场景

- 代码审查：委派给独立的 reviewer 角色
- 多语言翻译：委派给专门的翻译角色
- 多视角分析：不同角色使用不同模型从不同角度分析问题

### 配置要点

- 点击**添加角色**创建新的委派角色
- 每个自定义角色可独立配置模型、提示词和工具集
- 主角色可通过角色名称调用自定义角色完成任务
- 默认委派模式为同步调用

> 注意：当主角色占用本地 LLM 时，子角色只能复用相同模型，或改用 HF / 在线后端。

## 通用配置项

每个角色都支持以下配置项：

### 模型选择

| 配置项 | 说明 |
|--------|------|
| 后端类型 | 本地模型类别（LLM / 多模态 / HF 标准）或在线协议 |
| 模型 | 具体模型选择 |
| LoRA 挂载点 | 为本地模型附加 LoRA 微调适配器 |
| 在线提供方 | 在线模型的服务商 |
| API 凭据 | 绑定的 API Key |
| 局域网地址 | LAN 模式下的服务 URL |

### 提示词

| 配置项 | 说明 |
|--------|------|
| 角色提示词模板 | 预设的角色行为模板 |
| 系统提示词 | 自定义该角色的行为指令 |

### 核心参数

| 参数 | 说明 | 主角色默认值 |
|------|------|-------------|
| 启用工具 | 是否允许该角色调用工具 | 开启 |
| 温度 | 控制输出随机性（0~1） | 0.6 |
| 最大生成长度 | 单次回复最大 token 数（0=自动） | 0 |
| 上下文长度 | 单次任务最大上下文长度（0=无限制） | 空 |

> 注意：上下文长度为空时表示不覆盖模型库设定的上下文长度。推荐最低 32K

### 高级推理参数

| 参数 | 说明 | 主角色默认值 |
|------|------|-------------|
| 最大工具轮次 | 单次任务中工具调用的最大轮数 | 100 |
| 推理超时 | 单次推理最大等待秒数 | 600 |
| 快速模式 | 开启后工具调用前的过程反馈不进入下一轮上下文 | 关闭 |
| 思考模式 | 开启后支持模型生成原生思考块 | 开启 |

### 工具集配置

每个角色可独立选择允许使用的工具。支持两种模式：

- **自动选择**：运行时在已选工具范围内动态选择最合适的工具
- **固定加载**：始终加载全部已选工具

> 注意,任务执行时的可用工具完全遵循所选范围.包括自动选择或工具发现也不会调用未选的工具组
