## 自动审批

开启自动审批后，智能体执行需要确认的操作时将自动放行，无需手动干预。

### 开关控制

- 输入框上方有**自动审批开关**
- 默认关闭，会话独立(切换会话不会影响其他会话的开关状态)
- 开启/关闭时有提示消息确认

### 适用场景

| 场景 | 建议 |
|------|------|
| 信任的工作区 + 明确的任务 | 可开启自动审批提高效率 |
| 不熟悉的代码库 | 建议关闭，逐项确认 |
| 涉及系统配置的任务 | 建议关闭，避免意外 |
| 批量文件操作 | 可开启但建议先在测试工作区验证 |

### 自动审批行为

开启后，系统会：
- 自动允许命令执行（符合安全策略的）
- 自动允许文件写入操作
- 自动继续暂停的任务

**命令黑名单中的命令始终会被跳过**，即使在自动审批模式下也不会执行。

## 安全审查

系统对用户输入和工具输出进行**两层安全检测**。

### 第一层：输入检查

在用户消息进入智能体前，系统检查：

| 检测项 | 说明 |
|--------|------|
| 敏感词过滤 | 过滤不当内容 |
| 提示注入检测 | 防止通过用户输入越狱或修改智能体行为 |
| 结构破坏识别 | 检测 JSON/XML/代码注入、特殊令牌滥用 |
| 工具误用检测 | 防止调用非法工具或越界访问 |

### 第二层：输出检查

在工具执行结果返回前，系统检查：

| 检测项 | 说明 |
|--------|------|
| 恶意代码检测 | 检查执行结果中的危险代码 |
| 信息泄露识别 | 检测 API 密钥、私密信息 |
| 外部内容风险 | Web 爬虫、文档读取等外部内容的安全评估 |

### 安全暂停

检测到风险时，智能体进入**安全暂停**状态：

- 界面显示具体的风险原因
- 提供两个操作选项：
  - **继续**：确认允许该操作继续执行
  - **停止**：立即停止当前任务

## 暂停确认场景

以下情况智能体会暂停等待您确认：

### 命令执行确认

智能体准备执行终端命令时暂停，界面显示：

- 命令来源（终端命令）
- 工作目录
- 命令预览
- 风险提示（如需要安装依赖、修改文件、访问网络等）

操作选项：**允许** / **跳过**

### 工具轮次上限

达到当前工具调用轮次上限时暂停，可继续或停止。

### 重复执行警告

系统检测到当前路径可能陷入重复时暂停，提示确认是否继续。

### 模型停滞

智能体暂时没有获得新的有效进展时暂停，确认后继续尝试。

### 人工决策

智能体通过 `ask_human` 工具请求用户做出选择时，弹出决策对话框：

- 显示问题标题和选项
- 支持多组问题分页展示
- 支持**推荐选项**标记
- 支持**自定义输入**替代预设选项
- 提交后智能体基于您的选择继续执行

## 命令黑名单

命令黑名单是防止智能体执行危险命令的最后一道防线。

### 黑名单特性

- 支持**精确匹配**：如 `rm` 匹配 `rm`
- 支持**前缀匹配**：如 `rm` 可匹配 `rm file.txt`
- 支持**通配符**：`*` 和 `?`
- 自动覆盖参数变体：`rm` 一条规则即可阻止 `rm -rf /` 等变体
- 识别链式命令、管道、包装命令中的真实子命令

### 默认黑名单

系统预设了常用危险命令：`rm`、`del`、`format`、`shutdown`、`taskkill`、`dd`、`mkfs`、`git clean`、`git reset --hard` 等。

### 自定义黑名单

在高级参数 → 安全分组中可编辑命令黑名单，用逗号或换行分隔多个命令。

> ⚠️ 黑名单中的命令**始终被拒绝执行**，无论在自动审批模式还是手动确认模式下。

## 安全角色

安全角色是可选的角色组件，启用后方可执行上述安全审查。

### 启用方法

在右侧配置面板中，展开「角色参数」→「安全角色」→ 开启并配置模型。

### 与基础安全的关系

- 基础安全检查（敏感词过滤等）始终生效，无需安全角色
- 安全角色提供额外的**语义级**安全分析
- 建议处理高风险内容时启用安全角色
- 安全角色不指定模型时自动使用主角色模型
