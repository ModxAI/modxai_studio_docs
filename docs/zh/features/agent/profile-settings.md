> v1.1.0

指令设置用于配置全局指令文件和个人偏好，让智能体在每次执行时遵循您的个性化要求。

## 全局指令文件

指令文件是一个纯文本文件，其中的内容会作为系统指令注入到智能体的上下文中，影响智能体的行为模式。

### 配置方式

点击右侧面板的**指令设置**按钮，进入指令设置页面。

#### 方式一：自动检测（推荐）

1. 设置**工作区目录**
2. 系统会自动检测工作区根目录下的 `AGENTS.md` 类文件
3. 检测到后自动生效

#### 方式二：手动指定

1. 点击**浏览指令文件**
2. 选择一个纯文本文件

### 指令文件格式

- 纯文本文件（`.md`、`.txt` 等）
- 内容会作为系统提示词的一部分注入
- 建议包含项目规则、行为约束、输出格式等指令

## 个人偏好

个人偏好允许您添加多条简短描述，提醒智能体您的习惯和偏好。

### 添加偏好

1. 在指令设置页面找到「个人偏好」区域
2. 点击**添加偏好**
3. 输入描述（每条不超过 200 字）
4. 可添加多条偏好

### 偏好示例

- 「请始终使用中文回复」
- 「回复尽量简洁」

### 管理偏好

- 点击偏好条目旁的删除按钮移除
- 所有偏好会以较高权重合并注入到智能体的系统提示词中

## 工作区自动检测

设置工作区目录后，系统会在以下位置自动检测指令文件：

- 工作区根目录下的 `AGENTS.md`
- 工作区根目录下的 `.agents.md`
- 其他 AGENTS 类命名文件

检测到的文件会自动生效，无需手动指定。这是推荐的使用方式——只需在工作区根目录放置一个 `AGENTS.md` 文件，所有在该工作区中运行的智能体都会遵循其中的指令。
