# 定时任务

> v1.1.0
> 2026-05

定时任务允许您预设**智能体任务**、**记忆微调**或**知识库更新**的自动执行计划。任务可按固定间隔或指定时间自动触发，无需人工值守。

## 任务类型

定时任务支持三种类型：

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **智能体任务** | 按计划向智能体发送预设提示词并自动执行 | 定期生成报告、数据抓取、定时巡检 |
| **记忆微调** | 自动导出记忆并触发模型微调训练 | 定期基于新记忆优化个人模型 |
| **知识库更新** | 自动将记忆数据向量化并更新 RAG 知识库 | 周期性数据沉淀 |

### 智能体任务

按计划自动触发智能体执行指定任务。您可以配置：

| 配置项 | 必填 | 说明 |
|--------|------|------|
| 用户提示词 | ✅ | 要发送给智能体的任务指令 |
| 目标会话 | ✅ | **新会话**：每次执行新建会话 / **现有会话**：向已有会话发送消息 |
| 主模型（可选） | ❌ | 临时替换主角色模型，不覆盖会话原有配置 |

- **新建会话模式**：每次执行时创建全新会话，避免上下文干扰
- **现有会话模式**：向指定会话追加消息，适用于持续积累的对话场景。选择后需从下拉列表中选择已有会话

> **注意**：选择现有会话时，如果该会话当前正在运行智能体任务，本次执行会被阻塞，等待前序任务完成。

### 记忆微调

按计划自动执行记忆微调训练

| 配置项 | 必填 | 说明 |
|--------|------|------|
| 基础模型 | ✅ | 用于微调的 HF Standard 基座模型 |
| 训练模板 | ❌ | 选择训练模板预设 |
| 记忆分类 | ✅ | 数据范围：全部记忆 / 智能体记忆 / 收藏记忆 |
| 最小样本数 | ✅ | 触发训练所需的最低记忆样本数量（默认最低 10 条） |
| 迭代策略 | ✅ | **完全重建**：每次从头训练 / **持续迭代**：基于上次结果继续训练 |
| 输出目录 | ✅ | 训练结果存放目录 |

> **前置条件**：执行记忆微调需要内存中没有其他正在运行的智能体或训练任务。

### 知识库更新

按计划自动将记忆数据向量化并写入 RAG 知识库。需要：

| 配置项 | 必填 | 说明 |
|--------|------|------|
| 向量模型 | ✅ | 用于生成向量的 Embedding 模型，需先在模型库中导入 |
| 输出目录 | ✅ | 知识库数据输出目录 |
| 知识库名称 | ❌ | 默认 `modxai_rag_memory`，固定名称可保证多次更新累积到同一知识库 |
| 向量归一化 | ❌ | 是否对向量做归一化，默认开启 |
| 目标维度 | ❌ | 向量目标维度，默认 1024 |
| 池化策略 | ❌ | 向量池化方式：`mean` / `max` / `cls`，默认 `mean` |

> **前置条件**：如果未找到可用向量模型，请先在模型库中导入 Embedding 类型的模型。

## 调度策略

定时任务支持两种触发方式：

### 间隔触发

按固定时间间隔重复执行。配置：

| 参数 | 说明 | 可选值 |
|------|------|--------|
| 间隔数值 | 正整数 | 如 30 |
| 间隔单位 | 时间单位 | 分钟 / 小时 / 天 / 周 / 月 / 年 |

例如：「每 30 分钟」或「每 1 天」。

### 固定时间

在指定时间点执行一次。选择日期和时间即可。执行时间必须晚于当前时间。

## 创建与编辑任务

### 创建任务

1. 点击**新建任务**按钮
2. 填写任务名称（必填）
3. 从下拉列表中选择任务类型
4. 配置调度策略（间隔或固定时间）
5. 根据任务类型填写对应的任务配置
6. 点击确定完成创建

### 编辑任务

在任务列表中找到目标任务，点击**编辑**。修改参数后保存即可。编辑时：

- 如果修改了调度策略参数，系统会自动重新计算下一次执行时间

### 启用/停用

通过任务列表中的开关可随时启用或停用任务。停用后任务不会触发定时执行，但可手动触发。

## 任务列表与管理

任务列表展示所有已创建的定时任务：

| 列 | 说明 |
|----|------|
| 名称 | 任务名称 |
| 任务类型 | 智能体任务 / 记忆微调 / 知识库更新 |
| 触发方式 | 显示「每 X 分钟」或固定时间点 |
| 下次执行 | 预计的下次执行时间 |
| 最近状态 | 最近一次执行的结果状态 |
| 启用 | 切换开关 |

点击任务行可选中该任务，下方展开其运行历史。

## 运行历史

选中某个任务后，下方展示该任务的最近执行记录。

### 运行状态

| 状态 | 说明 |
|------|------|
| 排队中 | 任务已创建，等待执行 |
| 运行中 | 任务正在执行 |
| 成功 | 任务执行完成 |
| 失败 | 任务执行出错 |
| 阻塞 | 任务因资源占用等原因无法执行 |
| 已停止 | 被手动终止 |

### 触发来源

| 来源 | 说明 |
|------|------|
| 定时触发 | 按计划自动触发 |
| 手动触发 | 通过「立即执行」按钮触发 |
| 重试 | 系统自动重试 |

### 操作

- **查看运行详情**：显示该次运行的状态、触发来源、起止时间、错误摘要
- **停止运行**：手动终止正在执行的任务（仅运行中可用）
- **删除记录**：删除该条运行历史（运行中不可删除）
- **清除记录**：清除该任务下所有历史运行记录（运行中除外）

## 任务执行与停止

### 手动执行

点击任务行的**立即执行**按钮可手动触发一次任务，不受调度策略限制。执行结果会记录在运行历史中。

### 停止执行

如果任务正在运行（显示为「运行中」或「执行中」），可点击**停止当前任务**终止其执行。

### 并发限制

> **注意**：定时任务采用全局单并发策略 —— 同一时间只能有一个任务在执行。如果前一个任务尚未完成且新任务到达触发时间，新任务会被阻塞或跳过。

---

## 注意事项

- 记忆微调导出使用「记忆管理」中的导出能力，其导出兼容性规则与手动导出一致
- 知识库更新使用 `modxai_rag_memory` 作为默认集合名称，多次执行会对同一知识库**叠加更新**
- 定时任务的调度后台会定期扫描到期任务，15 秒扫描一次
- 如果 ModxAI 主服务重启，调度器会自动恢复未完成的任务状态
- 智能体任务在「现有会话」模式下，如果目标会话已删除，执行会失败
