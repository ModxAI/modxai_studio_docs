# 常见问题

> v1.1.0
> 2026-5

本文汇总了 ModxAI Studio 使用过程中的常见问题和解决方案。

---

## 快速入门

### 首次启动后应该做什么？

1. 进入**模型库**，下载或导入一个聊天模型
2. 加载模型后，进入**对话**页面即可开始使用
3. 如需使用高级功能（训练、数据处理等），请先在**设置**中安装环境

### 哪些功能需要在设置下载环境？

| 功能 | 环境要求 |
|------|---------|
| 文本对话（llama.cpp） | **无需** |
| 文本对话（HF Transformers） | **需要** CPU 或 GPU 环境 |
| 多模态对话 | **无需**（使用 llama.cpp） |
| 向量模型/重排模型 | **需要** CPU 或 GPU 环境 |
| SD 图像生成 | **需要** CPU 或 GPU 环境 |
| TTS 语音合成 | **需要** CPU 或 GPU 环境（推荐 GPU） |
| STT 语音转文本 | **需要** CPU 或 GPU 环境（推荐 GPU） |
| 模型训练 | **需要** GPU 环境（推荐） |
| 智能体（浏览器/Web工具） | **需要** 安装「全部」或「智能体」能力 |
| 音频转文本 | **需要** GPU 环境（推荐） |

## 模型管理

### 模型下载很慢怎么办？

- **切换下载源**：中国内地用户建议使用 ModelScope，海外用户使用 Hugging Face
- **检查网络**：确保网络连接稳定
- **分批下载**：大模型可选择部分量化版本先下载

### 导入模型后不显示？

- 点击**刷新按钮**更新模型列表
- 检查文件格式是否正确：
  - 聊天/多模态模型：`.gguf` 或 `.bin`
  - 向量/重排模型：`.safetensors`（需导入目录）

### 模型加载失败？

常见原因及解决方案：

| 原因 | 解决方案 |
|------|---------|
| 内存不足 | 关闭其他应用，或选择更小的量化版本（Q2_K、Q3_K_S） |
| 环境未安装 | 向量/重排/SD 模型需先在**设置**中安装环境 |
| 文件损坏 | 重新下载模型文件 |
| 格式不支持 | 确认文件格式符合要求 |

### LoRA 不兼容怎么办？

LoRA 必须与基础模型架构匹配。请确保：
- LoRA 是针对该基础模型训练的
- LoRA 格式正确（聊天模型使用 `.gguf` 格式的 LoRA）

### 同一时间能加载多少模型？

- **同类模型**：同一分类只能加载一个基础模型
- **不同类模型**：聊天、向量、重排、SD、TTS、STT 可同时加载
- **特别限制**：聊天和多模态不能同时加载

### HF Transformers 模型和 GGUF 模型有什么区别？

| 对比 | GGUF（llama.cpp） | HF Transformers |
|------|-------------------|-----------------|
| 环境要求 | 无需安装 | 需安装环境 |
| 推理速度 | 快 | 取决于设备 |
| 精度控制 | 量化版本固定 | 可灵活选择 float16/32 |
| 多模态 | 支持 | 不支持（仅文本） |
| 配额消耗 | 免费 | 加载扣配额 |

### 如何让其他应用调用我已加载的模型？

加载模型后，ModxAI Studio 自动提供 OpenAI 兼容的 HTTP API（`/v1/chat/completions`）。同一局域网内的其他设备或应用可直接调用。详见 [模型库 - 局域网推理服务](../features/models/overview.md#局域网推理服务)。

### 多模态模型能同时使用知识库（RAG）吗？

不能。多模态对话模式下 RAG 检索功能不可用。如需知识库问答，请切换到纯文本聊天模型。

## 智能体

### 智能体需要什么前置条件？

1. 为主角色加载一个聊天模型
2. 在**设置**中安装「全部」或「智能体」环境能力（需要浏览器和 Web 工具时）
3. 在对话界面切换到智能体模式

### 智能体适合多大的模型？

ModxAI Studio 的智能体框架专为本地小模型优化。1B-3B 参数的模型即可在 32K 上下文内完成简单的多步骤任务。更大的模型（7B+）可处理更复杂的任务。推荐使用多模态模型如:Qwen3.5

### 智能体能做什么？

- 自动搜索互联网获取信息
- 浏览和提取网页内容
- 读写本地文件
- 执行终端命令
- 管理任务清单并追踪进度
- 记住偏好

### 智能体的数据安全吗？

安全。智能体所有操作都在本地沙箱内执行，文件操作限制在授权目录，浏览器运行在隔离环境中。你的数据不会上传到任何云端服务。但如果使用在线模型需根据不同服务商协议要求自行判断.

## AI 对话

### 对话没有响应？

1. 确认模型已成功加载（状态显示"已加载"）
2. 检查页面底部的系统资源监控（内存/CPU 是否过载）
3. 尝试新建对话重试

### AI 回复为空或显示"模型未返回有效回复"？

建议：
- 重新表述问题，使描述更清晰
- 检查推理参数是否合理（温度不宜过高）
- 确认模型文件完整

### 多模态对话不显示附件按钮？

必须加载**多模态模型**才能使用图文对话功能。普通聊天模型不支持图片输入。

### 开启 RAG 推理后提示无法使用？

使用 RAG 功能前需要：
1. 先在**数据处理**模块创建 RAG 知识库
2. 在推理参数中选择知识库
3. 建议加载向量模型以及同时加载重排模型,但要注意资源占用.

### 图片格式限制？

- **支持**：JPG、PNG
- **不支持**：GIF、BMP、WebP 等格式

## 数据处理

### 处理过程可以切换页面吗？

可以。处理任务在后台运行，切换页面不影响进度。返回后可在"进度监控"查看状态。

### AI 节点处理很慢怎么办？

AI 节点需要调用模型推理，速度取决于硬件配置：
- 使用较小的模型进行处理
- 确保 GPU 可用以加速推理
- 先用少量数据测试效果

### 如何节省配额消耗？

- 合理设置数据拆分参数，避免生成过多碎片数据
- 关闭不必要的 AI 步骤（如 AI 过滤）
- 先用小数据集测试流程

### 断点续跑如何操作？

1. 取消勾选已完成的步骤
2. 保留需要重新执行的步骤
3. 确保前置步骤的输出数据存在

### 什么是结构化数据输入？

在 SFT 或 RAG 数据处理中，文件类型选择「JSON」即可进入结构化数据模式。适合已有固定字段的 JSON/JSONL 数据（如新闻、业务记录），可直接将字段映射到标准格式，无需走非结构化清洗链路。

### 导出格式 sft_chatml / pretrain / dpo 有什么区别？

| 导出格式 | 输出内容 | 适用场景 |
|----------|----------|----------|
| sft_chatml | system/user/assistant 对话格式 | SFT 微调训练 |
| pretrain | 纯文本语料 | 预训练 / 继续预训练 |
| dpo | prompt/chosen/rejected 偏好对 | DPO 偏好对齐训练 |

在AI合成步骤的导出格式参数中切换。预训练不再有独立步骤。

### 思维链模式（CoT）是什么？

开启后系统会自动注入推理型提示，引导模型输出思考过程。适合需要保留推理过程的数据.

## 模型训练

### 训练需要什么配置？

| 模型规模 | 显存需求（LoRA）| 显存需求（全量）|
|----------|----------------|----------------|
| 1B 以下 | 4GB | 16GB |
| 1-3B | 6GB | 24GB |
| 3-7B | 8GB | 40GB |
| 7-14B | 12GB | 80GB |

### "训练模型目录"填什么？

此参数需要指定 **HuggingFace 格式的模型目录**（包含 `.safetensors` 和 `config.json`），**不是** GGUF 推理模型。

### 检查点保存消耗配额吗？

是的，每保存一个检查点消耗 1 个训练配额。建议：
- 合理设置保存间隔（如每 100 步）
- 设置最大保存检查点数量

### 训练中断后如何恢复？

在训练准备页面，设置"恢复训练检查点路径"为上次保存的检查点目录。

## SD 图像生成

### 生成按钮点击没反应？

检查：
1. 是否已加载 SD 模型
2. 提示词是否为空
3. 是否有正在进行的生成任务

### 生成质量不理想？

尝试以下调整：
- 增加采样步数（推荐 30-50）
- 调整 CFG 强度（推荐 7-12）
- 使用更详细的提示词
- 添加负向提示词排除不想要的元素

### 图生图效果与原图差异太大？

降低**变化强度（Strength）**参数，推荐 0.3-0.6。

### 视频生成失败或卡住？

- 降低视频帧数（推荐 20-40 帧）
- 降低分辨率（推荐 512×512）
- 检查显存是否充足

### 资源占用建议

| 显存 | 推荐设置 |
|------|---------|
| 4GB | 512×512，步数 20-30 |
| 8GB | 768×768，步数 30-50 |
| 12GB+ | 1024×1024，步数 50+ |

## TTS 语音合成

### TTS 功能不可用？

检查以下几点：
1. 是否已在设置中安装环境
2. 环境安装后是否刷新了状态
3. TTS 能力是否显示为可用
4. 是否已加载 TTS 模型

### TTS 依赖安装很慢？

- misaki 库使用 PyPI 官方源安装，国内网络可能较慢
- 安装过程中请保持网络稳定
- 若超时失败，可通过重新安装环境重试

### TTS 能力显示不可用？

- TTS 依赖的 spaCy 模型是异步下载的，可能延迟数十秒至数分钟
- 主环境安装完成后，请稍等并刷新环境状态
- 若仍不可用，尝试重启应用或修复安装

### 生成的语音不自然？

- 检查输入文本是否有语法错误
- 尝试不同的音色预设
- 调整语速参数
- 确保语言设置与文本内容匹配

### TTS 只支持中英文？

是的，当前版本仅支持中文和英文。日语等其他语言需要额外依赖库，暂未内置。

### TTS 推理失败？

- 部分 AMD CPU 可能存在兼容性问题
- 建议使用 NVIDIA GPU 环境
- 检查模型是否正确加载

## 环境配置

### 环境安装失败？

常见解决方案：
- **切换镜像源**：中国内地用户选择阿里云或腾讯云
- **检查磁盘空间**：环境安装需要约 5-10GB 空间
- **检查网络**：确保能访问下载源
- **重试安装**：点击"重装"按钮

### CPU 和 GPU 环境选哪个？

- **有 NVIDIA 显卡**：选择 GPU 环境，推理和训练速度大幅提升
- **无独立显卡**：选择 CPU 环境，也可正常使用大部分功能

### 全部 / 智能体 / 语音合成 三种安装能力有什么区别？

| 安装能力 | 包含内容 | 适用场景 |
|----------|----------|----------|
| 全部 | 训练 + 智能体 + TTS 全部依赖 | 需要使用所有功能 |
| 智能体 | 核心库 + 浏览器环境 | 仅需智能体的网页和 Web 工具 |
| 语音合成 | 基础库 + TTS 依赖 | 仅需语音合成功能 |

### 智能体的浏览器工具无法使用？

需要安装「全部」或「智能体」安装能力。仅安装「语音合成」能力时，智能体的浏览器和 Web 工具不可用。

### TTS 安装后仍不可用？

在「全部」模式下，TTS 依赖为异步下载，完成后需等待一段时间再刷新环境状态。若异步下载失败，可切换到「语音合成」模式独立重装（同步下载）。

### 环境安装后需要重启吗？

不需要。安装完成后相关功能立即可用。但对于 TTS 异步安装，根据网络情况可能需要等待数分钟后刷新确认。

## 其他问题

### 数据存储在哪里？

- **模型文件**：保存在您指定的目录
- **对话记录**：存储在应用本地数据库
- **生成图片**：默认在系统临时目录，可在参数中指定保存位置

> 卸载应用不会删除本地数据库和模型文件。

### 如何完全清除数据？

需要手动删除：
- 模型文件目录
- 应用数据目录（C:\Users\<用户名>\AppData\Local\ModxAI）

### 如何反馈问题？

- **应用内反馈**：设置 → 帮助与反馈 → 提交反馈
- **邮箱**：support@modyu.tech
